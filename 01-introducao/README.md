# 01 - Introdução

Esta pasta registra o início dos meus estudos em Qualidade de Software e Teste de Software.

Nesta primeira etapa, comecei a compreender que a área de QA não se resume apenas a encontrar bugs, mas está diretamente ligada à prevenção de falhas, redução de riscos, melhoria da qualidade do produto e aumento da confiança na entrega final.

## Objetivo desta etapa

O objetivo deste módulo foi entender o papel do teste de software dentro do desenvolvimento de sistemas e iniciar a construção de uma base teórica para atuar na área de QA.

## Principais aprendizados

- Testar software é uma atividade essencial para avaliar a qualidade de um sistema.
- O teste ajuda a identificar defeitos antes que eles cheguem ao usuário final.
- Quanto mais cedo um problema é encontrado, menor tende a ser o custo para corrigi-lo.
- Teste não garante que um sistema esteja 100% livre de erros.
- A atividade de teste contribui para reduzir riscos e aumentar a confiança na entrega.

## Relação com QA

QA, ou Quality Assurance, está relacionado à garantia da qualidade do processo e do produto. Dentro desse contexto, o teste de software é uma das práticas utilizadas para verificar se o sistema atende aos requisitos, às expectativas do usuário e às necessidades do negócio.

## Minha percepção inicial

Nesta etapa, percebi que o profissional de QA precisa desenvolver uma visão crítica sobre o produto. Mais do que executar testes, é necessário entender o contexto, os riscos, os requisitos e os possíveis impactos de uma falha.

Também entendi que testar não é apenas uma atividade técnica, mas uma forma de proteger a experiência do usuário e o valor entregue pelo software.
