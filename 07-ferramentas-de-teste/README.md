# 07 - Ferramentas de Teste

Esta pasta documenta meus estudos sobre ferramentas utilizadas para escrita, manutenção, gerenciamento e execução de testes de software.

Nesta etapa, compreendi que ferramentas são importantes para organizar o processo de QA, mas que o mais importante é entender os conceitos por trás da documentação e execução dos testes.

## Objetivo desta etapa

O objetivo deste módulo foi conhecer algumas ferramentas utilizadas no dia a dia de QA e entender como elas apoiam a organização dos testes.

## Para que servem as ferramentas de teste?

As ferramentas de teste ajudam a organizar diferentes etapas do processo de qualidade.

Elas podem ser utilizadas para:

- Escrever casos de teste.
- Organizar suítes de teste.
- Gerenciar execuções.
- Registrar resultados.
- Acompanhar bugs.
- Gerar relatórios.
- Apoiar a comunicação entre QA, desenvolvimento e produto.

## Ferramentas estudadas

Durante esta etapa, foram citadas algumas ferramentas utilizadas no mercado:

| Ferramenta | Possível uso |
|---|---|
| Excel | Organização simples de casos de teste e evidências |
| TestLink | Gerenciamento de casos de teste |
| Zephyr | Gestão de testes integrada ao Jira |
| Azure Test Plans | Planejamento, execução e gestão de testes no Azure DevOps |
| Qase | Gerenciamento moderno de casos de teste |
| TestGear | Gestão de testes e relatórios |

## Excel

O Excel pode ser utilizado para criar e organizar casos de teste de forma simples.

Apesar de não ser uma ferramenta específica de QA, ainda pode ser útil em projetos pequenos, estudos ou quando a empresa não possui uma ferramenta dedicada.

## TestLink

O TestLink é uma ferramenta voltada ao gerenciamento de casos de teste.

Ele permite organizar suítes, planos de teste, execuções e resultados.

## Zephyr

O Zephyr é uma ferramenta muito utilizada em conjunto com o Jira.

Ele permite gerenciar casos de teste, ciclos de teste e execuções dentro do fluxo de trabalho de times ágeis.

## Azure Test Plans

O Azure Test Plans faz parte do ecossistema do Azure DevOps.

Ele permite planejar, executar e acompanhar testes, sendo bastante usado por equipes que já utilizam Azure DevOps para gestão de projetos e desenvolvimento.

## Qase

O Qase é uma ferramenta moderna para gerenciamento de testes.

Ela permite organizar casos de teste, execuções, suítes, relatórios e integrações com outras ferramentas.

## TestGear

O TestGear também é uma ferramenta voltada para gestão de testes, documentação e acompanhamento de resultados.

## O mais importante não é a ferramenta

Um ponto importante aprendido foi que as ferramentas costumam ter funcionamento parecido no dia a dia.

Por isso, para entrar na área de QA, não é obrigatório dominar profundamente todas as ferramentas existentes.

É mais importante saber:

- Escrever bons casos de teste.
- Criar cenários claros.
- Organizar evidências.
- Reportar bugs corretamente.
- Entender o ciclo de vida dos testes.
- Comunicar riscos e resultados.
- Adaptar-se às ferramentas utilizadas pela empresa.

## Minha percepção

Entendi que ferramentas são meios para organizar o trabalho, mas não substituem o raciocínio de QA.

Um bom profissional precisa primeiro entender o que testar, por que testar, como documentar e como comunicar os resultados.

Depois disso, aprender uma ferramenta específica se torna muito mais fácil.

## Conclusão

Nesta etapa, compreendi que existem diversas ferramentas para apoiar o trabalho de QA.

Também entendi que cada empresa pode utilizar uma ferramenta diferente, mas os conceitos fundamentais permanecem os mesmos.

Por isso, meu foco deve ser construir uma base sólida em documentação, casos de teste, reports de bugs e organização do processo de qualidade.
